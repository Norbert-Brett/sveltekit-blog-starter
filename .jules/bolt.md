## 2025-05-15 - Module-Level Promise Caching for Post Fetching
**Learning:** In SvelteKit (and Vite-based) applications, `import.meta.glob` with eager resolution or manual resolver calls can become a bottleneck when used repeatedly across different routes (e.g., blog index, RSS feed, API endpoints). Metadata parsing and sorting of file-based content on every request adds unnecessary overhead.
**Action:** Implement a module-level cache that stores the *Promise* of the processed data. This ensures the expensive disk I/O, parsing, and sorting logic runs exactly once per module lifecycle, even when multiple concurrent requests trigger the fetch for the first time. Shared utilities should be preferred over duplicating globbing logic to ensure consistent behavior and cache utilization.
