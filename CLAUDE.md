## Development Commands

All commands are run with [pnpm](https://pnpm.io/) (the project's package manager).

- **Start development server**: `pnpm dev`  
  Alias: `vp dev` (via vite-plus). Starts the Vite dev server at http://localhost:5173 with hot module replacement.

- **Build for production**: `pnpm build`  
  Alias: `vp build`. Generates a static site in the `build` directory using the `@sveltejs/adapter-static` adapter.

- **Preview production build**: `pnpm preview`  
  Alias: `vp preview`. Serves the built site locally from the `build` directory for testing.

- **Lint code**: `pnpm lint`  
  Runs ESLint on JavaScript, TypeScript, and Svelte files (respecting `.gitignore`).

- **Fix lint errors**: `pnpm format`  
  Runs ESLint with `--fix` to automatically correct linting issues.

- **Prepare/Vite-plus setup**: `pnpm prepare`  
  Alias: `vp config`. Initializes vite-plus configuration (typically run after installing dependencies).

## Project Architecture

### High-Level Structure

This is a SvelteKit 2 application configured as a static site generator (SSG) for a blog/portfolio. Key architectural decisions:

- **Routing**: File-based routing in `src/routes`. Each route corresponds to a page endpoint.
  - `src/routes/+layout.svelte`: Shared layout (includes header, footer, global styles).
  - `src/routes/+page.svelte`: Homepage.
  - `src/routes/about/`, `src/routes/contact/`: Static pages.
  - `src/routes/articles/`: Blog listing and individual post routes (`[post]` dynamic route).
  - `src/routes/api/`: JSON endpoints (posts, RSS feed).

- **Content**: Blog posts are Markdown files in `src/lib/posts`, processed by [mdsvex](https://mdsvex.compie.dev/) to allow Svelte components within Markdown.

- **Components**: Reusable components live in `src/lib/components`, organized into:
  - `ui/` – Generic UI elements (buttons, inputs, modals).
  - `magic/` – Interactive components (AI console, WebGL background, terminal).
  - `svg/` – Icon components.
  - Structural components (Header, Footer, PostsList, Pagination, etc.).

- **Styling**:
  - Global CSS in `static/css` (vanilla CSS base, designed for customization).
  - Tailwind CSS is configured via `vite-plus` (`@tailwindcss/vite`) for utility-first styling; see `tailwind.config.cjs` (if present) or the Tailwind directives in `static/css`.
  - Design tokens and guidelines are documented in `DESIGN.md`.

- **Static Assets**: Images, fonts, and files served from the root are placed in `static/`.

- **Configuration**: Site metadata (title, description, social links, navigation) is centralized in `src/lib/config.js`.

- **Data Layer**:
  - `src/lib/actions/` – Server actions for form handling (if any).
  - `src/lib/assets/` – Data files (e.g., testimonials, stats).

### Key Dependencies

- **Framework**: [SvelteKit](https://kit.svelte.dev/) + [Svelte 5](https://svelte.dev/blog/svelte-5-release)
- **Bundler**: [Vite](https://vitejs.dev/) via [vite-plus](https://github.com/voidzero-dev/vite-plus)
- **Markdown**: [mdsvex](https://mdsvex.compie.dev/) with [rehype](https://rehype.js.org/) plugins (slug, autolink headings)
- **UI**: [bits-ui](https://www.bits-ui.com/) (accessible UI primitives), [shadcn-svelte](https://www.shadcn-svelte.com/)
- **Animations**: [motion](https://motion.dev/), [gsap](https://greensock.com/gsap/), [lenis](https://github.com/studio-freight/lenis) (smooth scrolling)
- **Three.js**: For WebGL background (`three`, `@sveltejs/three`)
- **Linting**: [ESLint](https://eslint.org/) with [eslint-plugin-svelte](https://github.com/sveltejs/eslint-plugin-svelte)
- **Package Manager**: [pnpm](https://pnpm.io/) (version 11.4.0)

### Common Development Tasks

- **Adding a new page**: Create a route directory under `src/routes` (e.g., `src/routes/new-page/+page.svelte`).
- **Adding a blog post**: Drop a Markdown file with frontmatter (including `date` and `excerpt`) into `src/lib/posts`.
- **Customizing navigation**: Edit the `navItems` array in `src/lib/config.js`.
- **Updating site metadata**: Modify fields in `src/lib/config.js` (title, description, social URLs, etc.).
- **Styling adjustments**:
  - For global changes, edit `static/css`.
  - For component-specific styles, use Tailwind utility classes or component `<style>` blocks.
  - To add/remove Tailwind, follow the [official guide](https://tailwindcss.com/docs/guides/sveltekit) and adjust `static/css` accordingly.

### Notes

- The project is optimized for static site generation; dynamic server-side endpoints are limited to the `src/routes/api/` routes.
- There is no configured test suite (unit/integration) in this starter. Testing is typically done via manual inspection or by adding custom test configurations.
- The `build` directory is generated by `pnpm build` and contains the static output ready for deployment to any static host (Netlify, Vercel, GitHub Pages, etc.).
- Refer to `README.md` for detailed instructions on customization, deployment, and feature usage.
